The Computer can send several kinds of messages :
    - BUFF_SENDING : DataMessage
    - LIGHT_SENDING : AnswerMessage
    - BUFF_ASKING, asking for the buffer : RequestMessage
    - LIGHT_ASKING, asking for the light : RequestMessage
    - SCREEN_SIZE_ASKING, asking for the screen size : RequestMessage
    - FIRMWARE_VERSION_ASKING, asking for the firmware version : RequestMessage
    - FIRMWARE_UPDATE_SENDING, updating the firmware : DataMessage
    - ID_ASKING, asking for the product's ID : RequestMessage
    - AVAILABLE, asking if the device is available : RequestMessage

The cube can send several kinds of messages :
    - ACK_OK, ACK_NOK, ACK_ERR : AckMessage
    - LIGHT_RECEPTION : AnswerMessage
    - BUFF_RECEPTION : DataMessage
    - SCREEN_SIZE_RECEPTION, screen size : AnswerMessage
    - FIRMWARE_VERSION_RECEPTION, firmware version : AnswerMessage
    - ID_RECEPTION, product ID : AnswerMessage
    - ACK_OK, to say that it is available : AckMessage


DataMessage
          1 | ID_DEVICE(1) | OP_CODE(1) | SIZE_LEFT(2) | DATA(57) | CRC(2)
          0 | ID_DEVICE(1) | OP_CODE(1) | SIZE_LEFT(2) | DATA(57) | CRC(2)
          ...
          0 | ID_DEVICE(1) | OP_CODE(1) | SIZE_LEFT(2) | DATA(57) | CRC(2)

AckMessage
          1 | ID_DEVICE(1) | ACK_TYPE(1) | 0 | 3 | DATA(3) | CRC(2)

RequestMessage
          1 | ID_DEVICE(1) | OP_CODE(1) | 0 | 0 | CRC(2)

AnswerMessage
          1 | ID_DEVICE(1) | OP_CODE(1) | 0 | 1 | DATA(1) | CRC(2)
